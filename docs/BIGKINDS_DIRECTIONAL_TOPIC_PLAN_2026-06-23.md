# 빅카인즈 공개화면 기반 방향성 주제 개선 계획서

작성일: 2026-06-23

## 1. 목표

현재 경제 브리핑은 `confirmed_metrics`가 강해질수록 제목과 글감이 금리, 환율, 지수 같은 수치 분석형으로 기울어진다. 개선 목표는 수치를 주제의 출발점이 아니라 근거로만 쓰고, 빅카인즈 공개 화면에서 보이는 오늘의 이슈, 주간이슈, 키워드 흐름을 이용해 화자의 목적의식이 있는 주제를 먼저 고르는 것이다.

목표 흐름은 아래와 같다.

```text
빅카인즈 공개 이슈/키워드 수집
→ 사회적 관심 축과 산업 이슈 축 추출
→ 독자에게 전달할 메시지 선정
→ confirmed_metrics를 근거 역할로 매칭
→ 방향성 있는 제목과 본문 생성
```

## 2. 적용 범위

사용 대상은 유료 API가 아니라 공개 화면이다.

- 빅카인즈 서비스 안내: https://www.kinds.or.kr/v2/intro/service.do
- 주간이슈 공개 화면: https://www.kinds.or.kr/v2/news/weekendNews.do
- 메인/오늘의 이슈 공개 화면: https://www.kinds.or.kr/

수집 대상은 기사 본문이 아니다.

- 허용: 이슈명, 키워드, 분야, 기사 건수, 날짜, 공개 링크, 화면에 표시되는 짧은 메타정보
- 제외: 기사 본문 저장, 기사 전문 복제, 로그인 필요 데이터, 다운로드 파일 대량 수집, 유료 API 우회

## 3. 왜 빅카인즈가 필요한가

빅카인즈는 매일 수집된 뉴스의 주요 이슈와 키워드를 보여주며, 오늘의 이슈는 뉴스 클러스터링으로 상위 이슈를 구성한다. 오늘의 키워드는 정치, 경제, 사회, 국제, IT 과학 등 분야별 주요 개체명 흐름을 보여준다.

따라서 시장 수치만으로는 잡기 어려운 아래 질문에 답할 수 있다.

- 오늘 사람들이 실제로 관심을 두는 경제/산업 이슈는 무엇인가?
- 특정 숫자가 어떤 사회적 맥락과 연결되는가?
- 독자에게 “무엇을 보라”고 말할 수 있는가?
- 오늘 글은 수치 해설형인가, 리스크 방어형인가, 판단 순서형인가?

## 4. 핵심 설계

### 4.1 `BigKindsPublicCollector`

공개 화면에서 방향성 신호만 수집한다.

수집 필드:

- `issue_title`: 이슈명
- `category`: 경제, 국제, IT 과학 등
- `news_count`: 화면에 표시되는 기사 건수
- `keywords`: 화면에 표시되는 키워드 또는 개체명
- `source_url`: 빅카인즈 공개 화면 링크
- `collected_at`: 수집 시각
- `confidence`: 화면 파싱 성공도

운영 방식:

- 하루 2~4회 이하 저빈도 수집
- 6~12시간 캐시
- 실패 시 기존 시장 수치/RSS/GDELT 흐름으로 자동 후퇴
- HTML 구조 변경에 대비해 선택자 여러 개와 텍스트 패턴 fallback 준비

### 4.2 `DirectionalTopicPlanner`

빅카인즈 이슈와 시장 수치를 분리해서 다룬다.

입력:

- `bigkinds_issues`
- `market_snapshot`
- `source_pack.confirmed_metrics`
- 스케줄 슬롯: 국장전, 미장전, 글로벌, 휴장 대체

출력:

- `reader_problem`: 독자가 헷갈리는 지점
- `speaker_purpose`: 오늘 글이 전달할 한 문장
- `angle`: 판단 순서형, 리스크 방어형, 산업 흐름형, 공부형
- `evidence_roles`: 각 수치가 어떤 근거 역할을 하는지
- `title_candidates`: 수치 나열을 피한 제목 후보
- `rejected_titles`: 수치 중심 제목 탈락 사유

예시:

```text
빅카인즈 이슈: AI 반도체 투자, 삼성전자, 전력 수요
confirmed_metrics: KOSPI, US10Y, USD/JPY, BTC

speaker_purpose:
오늘은 반도체를 맞히기보다 AI 기대가 실제 수요와 투자로 이어지는 조건을 확인해야 한다.

title:
오늘 국장, 반도체를 맞히기보다 AI 수요가 숫자로 확인되는지를 보자

evidence_roles:
US10Y → 성장주 부담을 확인하는 보조 근거
USD/JPY → 아시아 위험 선호를 확인하는 보조 근거
KOSPI → 국내 시장 기준점
```

### 4.3 제목 품질 게이트

제목이 다시 수치 분석형으로 돌아가지 않도록 별도 게이트를 둔다.

감점 조건:

- 제목에 수치, 퍼센트, 금리, 환율, 지수명이 2개 이상 들어감
- 제목의 주어가 지표명으로 시작함
- “남긴 기준”, “보여준 신호”처럼 추상적인 수치 해석 표현만 있음
- 독자가 무엇을 판단해야 하는지 드러나지 않음

가점 조건:

- 독자의 행동이나 판단 순서가 보임
- 산업/생활/투자 습관과 연결됨
- 수치는 제목 밖으로 빠지고 본문 근거로 배치됨
- 반대 신호 또는 리스크 확인 방향이 포함됨

## 5. 구현 단계

### 1단계: 공개 화면 접근성 검증

목표: 빅카인즈 공개 화면을 자동 수집해도 되는 최소 범위와 기술 방식을 확인한다.

작업:

- 공개 화면 HTML에서 실제 이슈 데이터가 서버 렌더링인지 JS 렌더링인지 확인
- `robots.txt`, 이용약관, 저작권 문구 확인
- 기사 본문 수집 없이 이슈명/키워드/건수만 쓰는 정책 확정
- Playwright가 필요한지, 단순 HTTP fetch로 가능한지 결정

완료 기준:

- 공개 수집 가능한 필드 목록 확정
- 수집 불가 또는 구조 변경 시 fallback 규칙 확정

### 2단계: `BigKindsPublicCollector` 구현

목표: 빅카인즈 공개 화면에서 방향성 후보를 구조화한다.

작업:

- `modules/collectors/bigkinds_public.py` 추가
- `BigKindsIssue` 데이터 모델 추가
- 주간이슈/오늘의 이슈/오늘의 키워드 중 안정적으로 읽히는 화면부터 적용
- 캐시 파일 또는 DB 저장으로 반복 접속 최소화
- 기사 본문 저장 금지 테스트 추가

완료 기준:

- 네트워크 실패 시 프로그램이 멈추지 않음
- 최소 1개 이상의 방향성 이슈 후보가 생성됨
- 수집 결과가 `seo_snapshot` 또는 `market_snapshot` 보조 컨텍스트에 들어감

### 3단계: `DirectionalTopicPlanner` 구현

목표: 수치보다 메시지가 먼저인 제목 후보를 만든다.

작업:

- `modules/market/directional_topic_planner.py` 추가
- 빅카인즈 이슈와 `confirmed_metrics`를 매칭
- `speaker_purpose`와 `evidence_roles` 생성
- 수치 중심 제목을 탈락시키는 규칙 추가

완료 기준:

- 같은 수치가 들어와도 제목이 지표 나열형으로 치우치지 않음
- 제목에 독자 행동/판단 기준이 드러남

### 4단계: 생성 파이프라인 연결

목표: 실제 블로그 초안 생성 전에 방향성 주제가 먼저 들어가게 한다.

작업:

- 스케줄러 시드 생성 단계에서 빅카인즈 방향성 후보를 조회
- `ContentGenerator`의 시장 스냅샷 컨텍스트 앞에 `editorial_intent` 삽입
- Source Pack은 기존처럼 근거 검증 담당
- 발행 전 preflight에서 제목 품질 게이트 실행

완료 기준:

- 텔레그램 승인 요청 메시지에 “방향성 주제”와 “근거 수치”가 분리되어 보임
- 수치가 부족해도 글의 메시지가 유지됨

### 5단계: 운영 모니터링

목표: 공개 화면 변경이나 과도한 접속 리스크를 줄인다.

작업:

- 수집 성공률, 파싱 실패율, fallback 발생률 기록
- 빅카인즈 수집 실패 시 텔레그램에 비치명 알림
- 7일간 제목 품질 샘플링
- 수치 중심 제목 비율을 지표화

완료 기준:

- 빅카인즈 수집 실패가 발행 실패로 이어지지 않음
- 제목이 “수치 분석형”에서 “판단 기준형”으로 이동했는지 확인 가능

## 6. 다음 진행 사항

우선순위는 아래 순서가 좋다.

1. 빅카인즈 공개 화면 접근성 점검
2. 수집 가능 필드 확정
3. `BigKindsPublicCollector` 최소 구현
4. `DirectionalTopicPlanner` 최소 구현
5. 제목 품질 게이트 추가
6. 텔레그램 승인 메시지에 방향성/근거 분리 표시
7. 7일간 결과 모니터링 후 규칙 보정

## 7. 리스크와 대응

### 공개 화면 구조 변경

대응:

- 파서 선택자를 여러 단계로 둔다.
- 실패 시 기존 RSS/GDELT/시장 수치 흐름으로 후퇴한다.
- 실패를 치명 오류로 처리하지 않는다.

### 저작권/전재 리스크

대응:

- 기사 본문은 수집하지 않는다.
- 이슈명, 키워드, 건수, 링크만 저장한다.
- 블로그 본문에는 빅카인즈 내용을 그대로 옮기지 않고 “이슈 흐름”으로만 반영한다.

### 과도한 요청

대응:

- 하루 2~4회 이하로 제한한다.
- 캐시를 사용한다.
- 수동 재수집 버튼에도 쿨다운을 둔다.

### 방향성 과잉

대응:

- 빅카인즈 이슈가 시장 수치와 연결되지 않으면 경제 브리핑 제목으로 쓰지 않는다.
- `confirmed_metrics`는 여전히 본문 근거와 발행 안전장치로 유지한다.

## 8. 기대 효과

- 제목이 금리/환율/지수 나열에서 벗어난다.
- 글의 첫 문장이 “오늘 무엇을 볼 것인가”로 시작된다.
- 수치는 메시지를 보강하는 근거로 정리된다.
- 초보 독자가 읽었을 때 “그래서 오늘 내가 뭘 확인하면 되지?”가 더 분명해진다.
